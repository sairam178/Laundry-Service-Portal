The Problem

Why SudsFlow?

📞

Manual Booking

Customers had to call laundry shops, leading to errors, missed orders, and zero transparency.

📦

No Order Tracking

Once clothes were handed over, customers had no visibility into where their order stood.

💸

Cash-Only Payments

No digital payment options, making it inconvenient and difficult to track revenue.

📋

No Centralized Management

Admins had no unified platform to manage services, orders, or monitor revenue in real time.

Easy Booking

Security

Security & Data Integrity

🔒

bcrypt Password Hashing

Passwords hashed before storage. API never returns password hashes in any response.

🎟️

JWT Authentication

Every protected route validates a Bearer token carrying user ID and role.

🛡️

Role-Based Access Control

Middleware authorize() guards every route — customer, provider, and admin routes are completely separated.

💰

Server-Side Price Calculation

Order totals recalculated from DB at booking time — clients cannot tamper with prices.

📜

Full Status Audit Trail

Every status change appended to Order.statusHistory with timestamps.

🚫

No Self-Elevation to Admin

Registration only allows customer/provider. Admin role assigned directly in MongoDB only.
